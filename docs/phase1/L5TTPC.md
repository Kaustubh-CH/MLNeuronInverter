# L5TTPC fidelity backlog

Audit-driven list of remaining fidelity gaps between our jaxley L5TTPC port
(`toolbox/jaxley_cells/l5ttpc.py`) and the BBP `L5_TTPC1_cADpyr232_1` reference
(NEURON `biophysics.hoc` + `morphology.hoc`).

Companion to the runtime plan in `L5TTPC_runtime_plan.md`. Each item below is a
named, scoped follow-up — not in scope for the current branch.

## Just-shipped

### Apical Ih distance-dependent gradient — **partial**

`_apply_apical_ih_gradient` is called at build time (l5ttpc.py:_build), so
`cell.get_parameters()` and `cell.apical.nodes["Ih_gbar"]` reflect the BBP
formula

    gIh(d) = max(0, _IH_A + _IH_B * exp(d * _IH_K)) * _IH_BASE
           = max(0, -0.8696 + 2.087 * exp(d * 0.0031)) * 8e-5  (S/cm²)

at default state. **However**, jaxley v0.13's
`cell.apical.make_trainable("Ih_gbar")` collapses the per-compartment values
into a single mean scalar, and the bridge (`JaxleyBridge._build_handle`) then
`jnp.broadcast_to(...)`s the CNN-predicted scalar back across the apical
group at integrate time. Net effect during CNN-driven training/inference:
**apical Ih_gbar is uniform = CNN prediction** — gradient is overridden.

This matches `sim_jaxley_L5TTPC1.py`'s CSV-override path
(line 308–311), which is the canonical reference in the Neuron_Jaxley repo, so
correctness vs the reference is preserved. The gradient is preserved at default
state and during direct `jx.integrate(cell, ...)` calls without `params=`.

#### What would actually preserve the gradient under CNN training

Two viable approaches (both require touching `toolbox/JaxleyBridge.py`):

1. **Per-compartment trainables + scale broadcast.** Replace
   `cell.apical.make_trainable("Ih_gbar")` with one
   `cell.branch(b).comp(j).make_trainable("Ih_gbar")` per apical compartment.
   Extend `entry_to_cnn_idx` from `list[int]` to `list[(cnn_idx, scale)]` so
   the bridge does
   `val = flat_phys[cnn_idx] * scale` per entry. `scale[i,j]` =
   `max(0, _IH_A + _IH_B * exp(d[i,j] * _IH_K))` — pre-computed at build time.
   Net effect: CNN scalar acts as a multiplicative override on `_IH_BASE`,
   gradient shape preserved.

2. **Drop apical Ih from CNN-driven trainables entirely.** Apply gradient at
   build time, do **not** call `make_trainable` on apical Ih_gbar. CNN output
   index 3 (`gIhbar_Ih_dend`) only drives basal uniformly. Loses one CNN
   degree of freedom but exactly matches BBP biophysics.hoc when no CSV
   override is present.

(2) is cheaper to implement; (1) is more faithful to how the existing
training data was generated by `DL4neurons2` if it actually used CSV overrides.
Confirm DL4neurons2 path before choosing.

## Open fidelity items

### 1. dt = 0.1 ms vs NEURON's dt = 0.025 ms

`l5ttpc.py:_DT = 0.1`. NEURON BBP simulations and the `Neuron_Jaxley`
reference traces run at dt = 0.025 ms. At dt = 0.1, bwd_euler typically
drifts spike timing by 0.5–2 ms over a 500 ms trace. Our default-state
forward currently shows max|Δ| ≈ 94 mV, rmse ≈ 8.3 mV vs the
`results_detailed_morphology/jaxley_5k50kInterChaoticB_L5_sample_0.npz`
reference, with 2 spikes vs 4 in the reference. Most of this is plausibly
dt-driven; the Ih item above contributes too.

Fix is mechanical (`_DT = 0.025`) but quadruples integration cost — see
`L5TTPC_runtime_plan.md` for the trade.

### 2. nseg vs ncomp distribution

`_NCOMP = 4` is uniform across all branches. NEURON BBP cells use the
`d_lambda` rule with `lambda_f = 100 Hz`, which yields ~1–3 compartments
on short basal branches and 5–10 on long apical trunk segments. `audit_fixed_jaxley.py`
section 3 (line 147+) has the per-region nseg distribution if a per-branch
ncomp ever lands in jaxley.

### 3. Temperature (Q10) verification

NEURON sets `h.celsius = 34` and channel `.mod` files apply Q10 corrections
(typically Q10 = 2.3 for HH-style channels, 3.0 for Ca channels). Our
`bbp_channels_jaxley.py` ports must hard-code the same temperature factors —
verify by spot-checking each channel class against its `.mod` source in
`/pscratch/sd/k/ktub1999/Neuron_Jaxley/mechanisms/`. A 10°C off hard-codes to a
~10–30 % error in spike width.

### 4. e_pas and pas channel distribution

`BBPLeak_eLeak = -75.0` is set globally. `biophysics.hoc` may override per
region (axon often uses `-90.0` for fast-spiking variants; not sure for L5TTPC1
specifically). Confirm against the cell's `biophysics.hoc` in
`/global/cfs/cdirs/m3513/M1_Hoc_template/HocTemplate/L5_TTPC1_cADpyr232_1/`.

### 5. Stim injection site

`_attach_stim` injects at `cell.soma.comp(0).data_stimulate(...)`. Confirm
matches DL4neurons2's stim site — usually `soma(0.5)` (matches comp 0 with
ncomp=1 but not necessarily ncomp=4). For ncomp = 4, soma comp 0 is at
fractional position 1/8, not 0.5. Possibly negligible but worth checking
once dt and Ih fixes are deployed.

### 6. Apical capacitance

`cell.apical.set("capacitance", 2.0)` in `_build()`. BBP biophysics.hoc sets
`cm = 2` for apical and basal — looks correct, but a `try/except: pass`
swallows any failure silently. Replace with an assert once stable.

### 7. Region tagging via `assign_groups=True`

`jx.read_swc(..., assign_groups=True)` relies on SWC type codes (1=soma, 2=axon,
3=basal, 4=apical). Confirm `/pscratch/sd/k/ktub1999/Neuron_Jaxley/results_detailed_morphology/L5_TTPC1.swc`
has all four codes present and that no branch is mis-classified.
`audit_fixed_jaxley.py` section 6 covers this.
